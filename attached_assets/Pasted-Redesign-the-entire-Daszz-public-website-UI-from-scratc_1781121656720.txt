## New Visual Direction

Create a refined green **Digital Skin Twin Reveal**.

The ideal visual is a human face-inspired sculpture emerging softly from darkness.

Use this priority order:

1. Use an existing polished, optimized face-mask GLB only when the repository already includes a suitable high-quality asset.
2. Otherwise create an elegant procedural line-based face silhouette.
3. If procedural 3D does not look premium, use a layered 2.5D SVG face-contour composition with depth, parallax, soft lighting, and scan animation.

Do not force a poor-quality 3D result. The visual must look polished.

### Visual Details

Include:

* Symmetrical face silhouette
* Smooth jawline
* Forehead contour
* Cheek contours
* Subtle nose bridge
* Fine facial contour lines
* Thin scanning line moving slowly from top to bottom
* Very subtle green rim lighting
* Soft cream-green highlights
* Low-intensity background spotlight
* Slight pointer-follow parallax
* Gentle breathing-like motion
* Very small controlled rotation
* Subtle reveal progression tied to countdown percentage

The face should feel partially hidden and gradually revealed.

### Green Reveal Logic

As the countdown progresses:

* Reveal more facial contour lines
* Increase sculpture opacity slightly
* Increase soft green ambient glow subtly
* Shift scan-line position
* Never make the animation distracting

At launch completion, reveal the face fully and show:

```txt
DASZZ IS NOW LIVE
Your skin. Understood.
```

Performance requirements:

* Lazy-load the visual
* Use a polished fallback immediately
* Cap DPR around `[1, 1.5]`
* Reduce animation complexity on mobile
* Pause animation when the tab is inactive
* Respect `prefers-reduced-motion`
* Prevent layout shifts
* Keep the page smooth on mid-range Android devices
